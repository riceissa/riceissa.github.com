Bash Cheat Sheet

This document lists useful shortcuts when working in the Bash shell. This file is in alphabetical order.

Conventions

We will use the following conventions on this page in an effort to be concise.

Ctrl-⟨key⟩ — Hold the Control key and press the key ⟨key⟩. For instance, “Ctrl-a” would mean “hold the control key and press ‘a’ ”. When an upper case letter is necessary, we will write Shift. So “Ctrl-Shift-a” means “hold the control key and the shift key, and press ‘a’ ”. To avoid confusion, we will never write uppercase letters when describing these shortcuts, e.g. A.

Alt-⟨key⟩ — Hold the Alt key and press the key ⟨key⟩. The Alt key is sometimes called the Meta key. If the Alt key doesn’t work, try using the Escape key instead. (So Esc-⟨key⟩ instead. It may be necessary to release the Escape key before pressing ⟨key⟩.)

A smallword is any string of characters that contain only characters from the set {a–z,A–Z,0–9}. Smallwords are delimited by non-alphanumeric characters (whitespace, full stop, caret, etc.). This is similar to a lowercase “w” in the Vi(m) editor.

A bigword is any string of characters not containing a whitespace. Bigwords are delimited by whitespace characters (space, tab, etc.). This is similar to an uppercase “W” in the Vi(m) editor.

Control

Ctrl-a or Home — Jump to the beginning of the line.

Ctrl-b or Left-Arrow — Go back one character.

Ctrl-c — Terminate the command.

Ctrl-d — Delete the character under the cursor. If the line is empty, exit the terminal. This command fails if the cursor is at the end of a non-empty line.

Ctrl-e or End — Jump to the end of the line.

Ctrl-f or Right-Arrow — Go forward one character.

Ctrl-g — See [Ctrl-r].

Ctrl-h or Backspace — Delete the character to the left of the cursor.

Ctrl-k — Cut from under the cursor to the end of the line.

Ctrl-l — Clear the screen leaving the current line untouched.

Ctrl-m or Enter — Enter the command.

Ctrl-n or Down-Arrow — Show the next command in the history.

Ctrl-o — Accept the current line, then fetch the next line and reproduce it on the current line for editing. Consider the following:

# Comments will begin with “#”, like this line.
$ echo 1
1
$ echo 2
2
# Now press Ctrl-p twice, which fetches “echo 1”.
$ echo 1 # Now press Ctrl-o. The result should be:
1
$ echo 2 # This line is automatically produced.

Ctrl-p or [Up Arrow Key] — Show the previous command in the history.

Ctrl-q — Restore a frozen terminal. How do I freeze the terminal?

Ctrl-r — Reverse-search the history incrementally. For example, pressing Ctrl-r, then typing 'string', then pressing Ctrl-r, Ctrl-r, ..., will scroll through all commands in the history containing 'string'. Press Ctrl-g to break out of the search.

Ctrl-s — Freeze the terminal. How do I restore?

Ctrl-t — Swap the character under the cursor with the character to the left of the cursor. If this is used at the end of the line, swap the last two characters before the cursor.

Ctrl-u — Cut from the left side of the cursor to the beginning of the line.

Ctrl-v — Literally insert the next typed character. For example, [Ctrl-v][Tab] will insert a literal Tab character instead of completing the command.

Ctrl-w — Cut one bigword to the left.

Ctrl-xx — Jump between saved locations.
Pressing this will toggle between the current location and the beginning of the line. If you move the cursor after you have jumped to the beginning, then press this again, Bash will remember the location. Press this again to come back to the location you just saved.

Ctrl-y — Paste the most recently cut text.

Ctrl-x Ctrl-u or Ctrl-/ or Ctrl-_ — Undo the last operation.

Ctrl-x Ctrl-e — Open the default editor and execute the edited command. (This can be set with the $EDITOR variable, for example entering EDITOR=vim on a separate line will set the editor to Vim.)

Ctrl-z — Stop the current process. bg will restart it in the background and fg will restore it to the foreground.

Ctrl-] a — Move to the next occurrence of 'a'. How do I move to the previous occurrence of 'a'?

Ctrl-x / — Print filename completions.

Ctrl-x ~ — Print username completions.

Ctrl-x $ — Print shell variable completions.

Ctrl-x @ — Show host name completions.

Ctrl-x ! — Print command name completions.

Alt

Alt-b — Go back one smallword.

Alt-c — Capitalise the character under the cursor and move to the end of the current smallword.

Alt-d — Cut from under the cursor to the end of the current smallword.

Alt-f — Go forward one smallword. This does not work on the Gnome Terminal because Alt-f is bound to the "file" menu. However, [Esc][f] works.

Alt-l — Make the current smallword all lowercase (from under the cursor to the end of the sequence) and move the cursor to the end the sequence.

Alt-p — Reverse search the history non-incrementally.

Alt-r — Revert all changes to the current line. For example, if you edited a command from your history, this would revert to the original.

Alt-t — Swap the current smallword with the previous smallword.

Alt-u — Make the current smallword uppercase. (From under the cursor to the end of the current sequence.)

Alt-y — Rotate between the most recently cut text. (Only works if the last action was either Ctrl-y or Alt-y.)

Alt-Backspace — Cut to the beginning of the current smallword. (From left side of the cursor to the beginning of the smallword. Does not cut leading whitespace.)

Alt-< — Move to the first item in the history.

Alt-> — Move to the last item in the history (the current command, not the last executed command).

Alt-? — Show the current completion list.

Alt-* — Insert all possible completions.

Alt-/ — Attempt filename completion.

Alt-~ — Attempt username completion.

Alt-$ — Attempt shell variable completion.

Alt-@ — Attempt hostname completion.

Alt-! — Attempt command name completion.

Alt-& — Perform tilde expansion.

Alt-{ — Insert all filename completions within braces.

Alt-. or Alt-_ — Paste the last argument of the previous command. Press the same key combination multiple times to go further back in history.

Alt-Ctrl-] a — Move to the previous occurrence of 'a'. How do I move to the next occurrence of 'a'?

Alt-Ctrl-e — Expand the current line. For example, ~ will expand to one’s home directory. Expressions with relative history (e.g. !!), aliases, and shell variables (e.g. SHELL and TERM) will also be expanded.

Alt-Ctrl-y — Paste the second bigword of the previous command.

Other

Shift-[Page Up] — Scroll up one screen. (This may not work for some terminals. Use your mouse instead if you have one.)

Shift-[Page Down] — Scroll down one screen.

Working with Your History

The following commands are useful if you want to execute similar commands from your history.

!! — An alias for the previous command. (Useful for sudo !!, for example.)

!string — Execute the most recent command in the history beginning with 'string'.

!string:p — Print the most recent command in the history beginning with 'string', so you see what would have been run if you typed !string.

!n — Execute the nth command in the history, where n is a positive integer. Type history to get a list.

!$ — An alias for the last bigword of the previous command.

!^ — An alias for the first bigword of the previous command.

!* — An alias for the previous command without the last bigword. (So if the previous command was 'ls -A /home/$USER', then !* would give just 'ls -A'.)

!n:p — Print the nth command in the history. !$:p, !^:p, ..., also work.

^string1^string2 — In the previous command, replace the first occurrence of string1 with string2, then execute.
